## Overview

The goal is not simply to use AI. It is to design a repeatable workflow where AI output is reviewed, facts remain distinguishable from suggestions, and Git preserves the history of every public change.

## Workflow

```
Capture -> Clarify -> Build -> Store -> Publish -> Review
```

- **Capture:** screenshots, articles, ideas, and questions
- **Clarify:** expose assumptions and missing evidence
- **Build:** update Markdown and code with ChatGPT, Codex, and VS Code
- **Store:** preserve changes with Git and GitHub
- **Publish:** generate project pages and writing
- **Review:** verify facts, links, privacy, and build output

## Verification Rules

| Content | AI assistance | Manual verification |
|---|---|---|
| Engineering explanation | Structure and alternatives | Equations, units, assumptions, simulation results |
| Exchange information | Checklists and questions | Official dates, visa rules, university requirements |
| Website code | Implementation support | Build, links, accessibility, deployment |
| Personal profile | Editing and translation | Accuracy and privacy |

## Limitations

AI may produce fluent but unsupported claims, lose dates or units, and generate code that conflicts with an existing workflow. Time savings have not yet been measured, so no numerical productivity claim is made.

## Next Step

Add a content checklist for sources, update dates, privacy review, and broken links, then measure whether the workflow actually reduces repeated work.
